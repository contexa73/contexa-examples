What is @Protectable? @Protectable enables method-level Zero Trust protection. Each API call is evaluated by the AI engine. Async returns immediately while AI analyzes in background. Sync waits for AI decision before returning. ownerField binds the resource to its owner for ownership-based access control.
@Protectable Patterns

Pattern 1: Basic Async

ASYNC
GET /api/orders
findAll() - Returns immediately, AI analyzes in background. No owner binding.

Pattern 2: Async + ownerField

ASYNC ownerField
GET /api/orders/customer/{customerId}
findByCustomer() - Async with ownership check. Only the resource owner should access.

Pattern 3: Sync (Wait for AI)

SYNC
POST /api/orders
createOrder() - Waits for AI decision before processing. Compare response time with async.

Pattern 4: Sync + ownerField

SYNC ownerField
DELETE /api/orders/{orderId}?customerId=
deleteOrder() - Sync AI decision + ownership verification. Most restrictive pattern.
